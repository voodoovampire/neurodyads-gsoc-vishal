## Project overview

NeuroDyads studies **brain‑to‑brain decoding** using EEG hyperscanning data recorded from interacting dyads. Each dyad consists of a **Speaker** telling a story and a **Listener** listening to the story, while EEG is recorded simultaneously from both participants.

This repository contains:

- A complete notebook implementing the **pre‑task pipeline**:
  - Event‑based segmentation using DIN1 markers
  - Preprocessing and ICA‑based artifact removal
  - Power spectrum analysis before and after ICA
  - Construction of a combined Speaker–Listener representation
  - Training of a 3‑dimensional **CEBRA** embedding
  - Visualization of embeddings and temporal structure
  - Shuffled‑label control analysis
- Saved figures and metrics generated by the notebook
- Documentation files describing methodology and timeline

The aim is to provide a **reproducible, clearly documented baseline** that can be extended during GSoC into a more modular NeuroDyads analysis pipeline.

## Project goals

This repository has three main goals:

1. **Preprocessing baseline**  
   Build a clear and reproducible preprocessing pipeline for paired EEG recordings from a Speaker and a Listener.

2. **Representation learning baseline**  
   Learn a low‑dimensional joint embedding of both brains using **CEBRA** and visualize how task labels organize in this space.

3. **Control and evaluation baseline**  
   Implement shuffled‑label control analysis and simple metrics to check whether the embeddings capture meaningful structure beyond chance.

## Pipeline description

### 1. Loading and segmentation

- Raw EDF files are loaded with **MNE**.
- DIN1 markers are used to identify relevant events in the story task.
- Time windows are cut around these markers to create trials for both Speaker and Listener.
- Trials are aligned across participants to ensure they correspond to the same temporal segments.

### 2. Preprocessing and ICA

For each participant:

- Remove or ignore the VREF channel and any obviously bad channels.
- Apply band‑pass filtering (e.g. 1–40 Hz).
- Fit an **ICA** model to the continuous data.
- Visually inspect ICA components and remove artifact components (e.g. eye blinks).
- Reconstruct cleaned data and compute **power spectra before and after ICA**.

The figure `power_spectrum_before_and_after_ICA.png` in `results/figures/` summarizes this step.

### 3. Joint dyad representation

- For every aligned time window, Speaker and Listener channel data are combined into a **single joint representation** (conceptually a 128‑channel vector if each participant has 64 channels).
- These joint samples form the input matrix for CEBRA, with corresponding labels describing task or time information.

### 4. CEBRA embedding

- **CEBRA** is trained on the joint representation with task/time labels.
- The main embedding used here is 3‑dimensional for easier visualization.
- Embeddings are saved as `.npy` / `.npz` files in `results/metrics/`:
  - `neurodyad_cebra_time_3d.npz`
  - `neurodyad_cebra_time_embedding.npy`
- 3D scatter plots and time‑trajectory plots are saved in `results/figures/`, for example:
  - `3d_CEBRA_embedding_Training.png.png`
  - `cebra_time_embedding_trajectory.png`
  - `cebra_time_state_sequence_over_story.png`

### 5. Control analysis

To check that observed structure is not due to random labels:

- Labels are randomly shuffled.
- CEBRA is retrained using the shuffled labels.
- A 3D embedding is generated and saved as:
  - `3D_CEBRA_Training_suffled_training.png.png`
- Qualitative comparison between true‑label and shuffled‑label embeddings shows how much task information is preserved.
- Quantitative metrics (e.g. simple KNN accuracy) are stored in `results/metrics/` via `neurodyad_windows_v1_metadata.csv` and related files.
